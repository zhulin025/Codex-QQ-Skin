# Codex Token 统计与 QQ 等级系统规划

状态：已于 `2.4.0` 实施
目标版本：`2.4.0`

## 1. 目标

在 QQ 皮肤右侧栏加入一个完全本地的“Codex 成长中心”：

- 默认替换右侧上方现有的原生输出/来源区域，直观显示 Codex token 数据；
- 保留下方现有“Codex 伙伴”卡片和宠物交互；
- 根据活跃天数与总 token 计算成长值和等级；
- 使用 QQ 经典的星星、月亮、太阳、皇冠图标表达等级；
- 不要求注册额外账号，不读取或上传 API Key，不上传 token 记录；
- 不明显增加 Codex 启动时间和常驻资源占用。

本功能统计的是**本机 Codex 使用情况**，不是 OpenAI 官方账单或套餐额度。

## 2. 已确认的现状

### 2.1 本地日志解析能力

项目内置 Codex 本地日志解析器：

- 同时扫描 `~/.codex/sessions` 和 `~/.codex/archived_sessions`；
- 读取 `event_msg/token_count`，区分输入、输出、缓存输入和推理输出；
- 按 30 分钟生成 bucket，并生成 session 活跃数据；
- 已处理归档移动、重复文件、重复 token 事件；
- 已处理 fork 和子 Agent 继承父任务历史导致的重复计数；
- 使用可丢弃的增量缓存，热更新不需要每次重扫所有历史日志。

成长中心只使用 Codex 本地日志，不接入在线 API 或其他工具的汇总数据。

### 2.2 QQ 皮肤当前结构

右侧栏目前分为两块：

1. 上方是 Codex 原生 `[data-pip-obstacle="thread-summary-panel"]`，包含输出和来源；
2. 下方是皮肤注入的 `#codex-qq-skin-companion`，包含宠物、快捷操作、任务状态与本周额度。

注入器运行在 Node.js 层，通过本机 CDP 把 CSS、图片和渲染脚本注入 Codex。这个 Node 层可以安全读取本地日志；浏览器渲染层不应该直接取得文件系统权限。

## 3. 推荐界面

### 3.1 右侧栏结构

默认布局：

```text
│ 头像  Lv.12  ☾☾☾          │
│ 距离 Lv.13 还差 34 成长值   │
│ ███████████░░░  72%        │
├────────────────────────────┤
│ 今日        本周       累计 │
│ 428K       2.6M       38M  │
│ Token      Token      Token│
├────────────────────────────┤
│ 近 7 天  ▁▃▂▅▇▄▆           │
│ 活跃 18 天 · 连续 6 天      │
│ 输入 21M / 输出 4M / 推理 3M│

└────────────────────────────┘
┌────────────────────────────┐
│ Codex 伙伴                  │

│ 在线 · 随时待命  本周剩余80%│

```

### 3.2 原生面板处理

- 默认显示“成长中心”，隐藏原生输出/来源内容；
- 标题栏保留一个“资料”按钮，点击后临时切回原生输出/来源；
- 原生 DOM 不删除，避免影响 Codex 的文件、来源和面板状态；
- 用户的选择保存在本地：`统计` / `资料`；
- 快速聊天、设置页、窄窗口继续沿用当前的隐藏规则；
- 窗口低于三栏阈值时不强行显示右栏，保持现有响应式行为。

### 3.3 首屏只展示的指标

- 今日总 token（默认包含缓存）；
- 最近 7 天总 token（默认包含缓存）；
- 历史累计总 token（默认包含缓存）；
- 活跃天数与连续活跃天数；
- 当前等级、等级图标、升级进度；
- 近 7 天迷你柱状图。

输入、输出、推理、缓存等细项放在卡片下半区或详情态，不在首屏堆满数字。

## 4. 数据口径

### 4.1 Token 定义

统计采用非重叠字段，并提供两种显示口径：

```text
净用量 = 非缓存输入 + 普通输出 + 推理输出
总用量 = 净用量 + cachedInputTokens
```

界面默认展示更直观、更有成就感的“总用量”；等级行右侧提供“净用量”开关，开启后今日、近 7 天、历史累计和趋势图统一排除缓存。Codex 的原始 `input_tokens` 包含缓存输入、`output_tokens` 包含推理输出，因此聚合时仍先拆成非重叠字段，再计算两种口径，避免重复相加。

### 4.2 时间范围

- 今日：本机当前时区自然日；
- 本周：最近 7 个自然日，不使用套餐重置周期；
- 累计：本机可发现的 live + archived Codex 日志；
- 活跃日：当天存在 token，或从本版本开始检测到皮肤成功启动；
- 连续活跃：截至今天连续存在活跃记录的自然日数量。

“本周 token”与当前卡片中的“本周剩余额度”是两个不同概念，UI 必须分别命名，不能让用户误以为 token 数等于官方套餐消耗百分比。

### 4.3 暂不展示费用

首版不计算美元费用：Codex 可能使用订阅额度，不同模型价格也会变化，展示估算金额容易被误解为真实账单。

## 5. QQ 等级规则

### 5.1 图标换算

使用经典 QQ 的四进制视觉规则：

- 1 星星 = 1 级；
- 1 月亮 = 4 级；
- 1 太阳 = 16 级；
- 1 皇冠 = 64 级；
- 满 4 个低阶图标进 1 个高阶图标。

等级图标通过等级整数直接分解，例：`Lv.27 = 1 太阳 + 2 月亮 + 3 星星`。

### 5.2 每日成长值

每天最多获得 `2.0` 成长值：

| 当日行为 | 成长值 |
|---|---:|
| 当天启动皮肤或产生 Codex token | +1.00 |
| 总 token 达到 10K | +0.25 |
| 总 token 达到 100K | +0.50 |
| 总 token 达到 500K | +0.75 |
| 总 token 达到 2M | +1.00 |

token 奖励按达到的最高档计算，不叠加。这样能体现使用深度，又不会鼓励浪费 token 或让一次超大任务瞬间跳很多级。

### 5.3 等级计算

采用 QQ 早期等级成长曲线的形式：

```text
达到 Lv.N 所需累计成长值 = N² + 4N
```

示例：

| 等级 | 累计成长值 | 图标 |
|---:|---:|---|
| 1 | 5 | ★ |
| 4 | 32 | ☾ |
| 16 | 320 | ☀ |
| 32 | 1,152 | ☀☀ |
| 64 | 4,352 | ♛ |

升级只增长，不因删除日志或统计修复而降级。实现时保存 `highestLevel` 与 `highestGrowth`，重新计算值只允许向上修正；如未来确需重算，必须由用户主动执行。

### 5.4 历史初始化

- 首次启用时，根据已有 Codex token 日志回算历史活跃日与 token 档位；
- 本版本安装后的“只登录但未产生 token”由本地每日心跳补足；
- 没有历史日志的新用户从 0 级开始，完成若干活跃日后升到 1 级；
- 0 级显示一颗灰色空心星，不显示虚假的已获得等级。

### 5.5 防刷与公平性

- 缓存输入计入 token 成长档位，满足累计成就展示；
- fork、子 Agent 的继承历史和重复 token 事件必须去重；
- 同一天 token 奖励封顶 +1；
- 修改系统时间不补发未来日期成长值；
- 不引入排行榜，不上传等级，不把“消耗更多 token”描述成效率更高。

## 6. 技术架构

```text
~/.codex/sessions + archived_sessions
                │ 只读
                ▼
Codex usage worker（Node 子进程）
  - 解析 Codex 本地 session 日志
  - 去重、增量缓存、按日本地聚合
                │ 仅输出聚合 JSON
                ▼
QQ Skin injector（Node）
  - 校验 schema、大小和数值范围
  - 通过现有 loopback CDP 推送快照
                ▼
renderer-inject.js
  - 渲染成长中心
  - 不接触日志路径和文件内容
```

### 6.1 新增模块建议

```text
scripts/usage/codex-usage-worker.mjs   # 子进程入口与快照生成
scripts/usage/aggregate-usage.mjs      # 日/周/累计与等级计算
scripts/usage/level-rules.mjs          # 纯函数等级规则
scripts/usage/vendor/                  # 最小 Codex 解析模块
assets/renderer-inject.js              # 新增 usage snapshot 渲染
assets/qq-skin.css                     # 成长中心样式
tests/usage-worker.test.mjs
tests/level-rules.test.mjs
```

### 6.2 第三方代码管理

本地解析模块的维护要求：

- 只保留 Codex parser 及其必要依赖，不加入联网同步、登录、API Key 和其他工具 parser；
- 修改解析器后必须运行 Codex fixture 测试；
- 保持缓存格式可迁移，并对历史日志解析结果做回归校验。

### 6.3 刷新策略

- 皮肤先正常注入，统计在后台异步加载，绝不阻塞 Codex 首屏；
- 启动后生成一次快照；
- Codex 活跃时最多每 60 秒增量刷新一次；
- 用户可点击“刷新”；
- worker 设置超时，失败不能拖垮注入器；
- 冷启动建立索引时显示“正在建立本地统计索引”；
- 热缓存目标：刷新无明显 UI 卡顿；
- 统计快照限制在小型聚合 JSON，不把原始 prompt、路径或日志内容注入页面。

### 6.4 本地状态

建议使用：

```text
macOS:  ~/Library/Application Support/CodexQQSkin/usage/
Windows: %APPDATA%\CodexQQSkin\usage\
```

保存内容：

- parser 增量缓存；
- 最近一次聚合快照；
- 每日启动心跳；
- `highestGrowth` / `highestLevel`；
- 面板模式与最近更新时间。

macOS 文件权限设为 owner-only；Windows 使用当前用户应用数据目录。任何文件都不得包含 API Key、prompt 正文或完整对话。

## 7. 账号边界

目前 Codex rollout 的稳定元数据没有可靠的账号 ID，因此 token 历史无法准确按同一台电脑上的多个 Codex 账号拆分。

首版必须明确：

- 名称使用“本机 Codex 等级”和“本机累计”；
- 不把本机 token 数据绑定到当前头像或冒充账号云端等级；
- 当前已有的“本周剩余额度”继续按账号隔离缓存；
- 如果未来 Codex 日志提供稳定账号 ID，再迁移为账号级统计；
- 不允许用当前登录账号直接接管此前所有本机历史。

## 8. 状态与异常设计

| 状态 | 显示 |
|---|---|
| 首次扫描 | 正在建立本地统计索引… |
| 无日志 | 完成一次 Codex 任务后，这里会出现统计 |
| 正常 | 数值 + 更新时间 |
| 数据较旧 | 数据更新于 HH:mm，点击刷新 |
| 解析失败 | 暂时无法读取本地统计；宠物与皮肤仍正常工作 |
| 原生资料模式 | 显示 Codex 原有输出/来源，按钮可返回统计 |

错误详情只写入本地 injector 日志，UI 不展示本机绝对路径。

## 9. 测试与验收

### 9.1 数据正确性

- 输入、输出、推理和缓存字段不重复相加；
- live 与 archived 同一 session 不重复；
- fork 和子 Agent 的继承 token 不重复；
- 重复 `token_count` 不重复；
- 本地日界线、最近 7 天和连续活跃日正确；
- 日成长值上限为 2；
- 删除日志不会导致已获得等级下降。

### 9.2 等级边界

至少覆盖 `Lv.0/1/3/4/15/16/63/64`，验证四进制图标、升级阈值和进度百分比。

### 9.3 UI 回归

- 右栏宽度 272–360px 都无溢出；
- 资料/统计切换真实可点击；
- 原生输出和来源没有被删除；
- 宠物、终端、声音、本周剩余额度保持可用；
- 快速聊天、设置页、首页和窄窗口不出现悬浮遮挡；
- macOS 与 Windows 都使用同一份快照 schema。

### 9.4 性能验收

- 统计初始化不阻塞皮肤注入；
- 不在 renderer 中扫描文件或运行持续动画图表；
- 无数据变化时不重复推送 DOM；
- worker 超时或退出后，Codex 和皮肤继续可用；
- 大历史日志使用增量缓存，不进行每分钟全量重扫。

## 10. 实施顺序

### 阶段 A：数据层

1. 固定并裁剪 Codex 本地日志解析模块；
2. 定义 `usage-snapshot-v1` schema；
3. 实现日/周/累计、活跃日和 token 分项；
4. 实现异步 worker、缓存、超时和安全校验；
5. 用 Codex 日志 fixtures 做对照测试。

### 阶段 B：成长中心 UI

1. 新建上方统计卡片；
2. 隐藏但不删除原生输出/来源；
3. 增加“统计 / 资料”切换；
4. 加入 7 日迷你图、加载态、空态和错误态；
5. 回归三栏、快速聊天和设置页。

### 阶段 C：QQ 等级

1. 实现成长值和等级纯函数；
2. 实现星/月/太阳/皇冠图标组件；
3. 加入首次历史回算、每日心跳和等级只升不降保护；
4. 加入规则说明弹层；
5. 完成边界、迁移与防刷测试。

### 阶段 D：跨平台与发布前验证

1. macOS 实机热应用和长期运行验证；
2. Windows 打包、路径、权限和子进程验证；
3. 检查第三方许可证与 NOTICE；
4. 更新 README、CHANGELOG 和功能截图；
5. 本地构建并验证安装包。未经明确发布指令，不上传、不部署、不创建 Release。

## 11. 首版不做

- 多设备云同步；
- 用户排行榜和好友 PK；
- 第三方登录或 API Key 接入；
- OpenAI 账单金额推算；
- 多 Codex 账号历史自动归属；
- 因使用特定模型、并行 Agent 数量而额外加分；
- 用 token 数评价用户效率或工作质量。

## 12. 建议的最终产品定义

这不是一个“烧 token 排行榜”，而是一套本地、轻量、带 QQ 怀旧感的 Codex 使用足迹：

- token 数据让用户知道自己最近怎么使用 Codex；
- 活跃天数承担主要成长权重；
- token 只提供有封顶的使用深度加速；
- QQ 图标让长期使用有可见的积累感；
- 所有原始数据留在本机，统计失败也不影响 Codex 正常工作。
