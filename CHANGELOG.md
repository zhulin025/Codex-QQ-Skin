# Changelog

## 2.0.0 — 2026-07-20

- 原生 macOS App 新增“上传图片，生成我的皮肤”入口；用户选择图片并命名后即可自动生成并立即应用整套 QQ 皮肤。
- 图片只在本机处理：自动分析主色、亮度、视觉焦点、左右安全留白、画面比例与任务页呈现方式，不上传到任何服务器。
- 自定义主题不再写死默认强调色，确保图片分析得到的协调配色可以直接驱动标题、描边、卡片和背景材质。
- Windows 10/11 x64 包新增图片选择入口，与 macOS 共用同一套自适应皮肤引擎。
- 修复从“一键切换”停留在原生界面后上传图片会保存成功但注入验证失败的问题；明确的上传、应用或启动操作现在会重新启用皮肤，后台热更新仍尊重用户之后的切换选择。
- 重构为三套互相独立的皮肤模式：Codex 原生、固定 QQ 皮肤、用户自定义皮肤；右上角使用三段式切换器，上传成功后默认选择“自定义”。
- 用户图片不再进入 QQ 复古外框、三栏布局和伙伴卡，而是沿用 Codex 原生结构，参考 Dream Skin 的本地图像分析与首页构图规则生成沉浸式新建任务页。
- QQ 皮肤固定使用内置主题配置与 `portal-hero.png`，不会再被用户上传图片或自定义配色覆盖。
- 人物及普通照片在新建任务顶部构建框中改用完整适配，避免横幅比例裁掉头部或身体；超宽背景图仍使用铺满模式。
- 修复从自定义皮肤切回 QQ 时只恢复外框的问题：现在会重新读取 QQ 的固定蓝色配色和三栏布局，并恢复左栏、右侧摘要托盘与 Codex 伙伴面板。
- 兼容新版 Codex 将固定摘要开关重命名为“显示/隐藏侧边栏”的 DOM，使用真实的右上角坐标区分左侧导航开关。

- 菜单栏与 `status-qq-skin-macos.sh` 不再依赖 `/usr/bin/python3`（macOS 12.3+ 默认不预装）读取主题名与运行状态，改用纯 shell 解析；此前在未装 Xcode 命令行工具的机器上，主题名会退化成 id、`--json` 状态直接失效

### 安全

- CDP 端点必须由已验证的官方 Codex 可执行文件或其子进程监听；WebSocket 还会校验 loopback、page ID、路径、无重定向，并安全处理畸形消息和发送异常。
- 主题配置与图片使用真实路径 containment，拒绝 symlink 越界、空文件、超过 16 MB、单边超过 16384 px 或超过 50 MP 的图片；主题展示文本拒绝换行和控制字符。
- AppleScript 动态内容全部通过 argv 传递；SwiftBar 过滤主题 ID、文件名和菜单文本，避免主题元数据改变菜单属性或命令参数。
- `config.toml` 只按严格 UTF-8 读取，拒绝 NUL、歧义多行 TOML、重复 `[desktop]`，通过用户级锁、原始字节核验和同目录原子替换保护中文配置与并发写入。
- 暂停和恢复会在 TERM/KILL 前后核验 watcher 的 PID、启动时间、Node 和 injector 路径；不匹配的进程绝不结束，已确认 watcher 若无法停止则保留 state 并中止，不再报告假成功。
